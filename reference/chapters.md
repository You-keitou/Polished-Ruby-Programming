# Polished Ruby Programming - 研鑽Rubyプログラミング 章構成

## 第I部 Rubyプログラミングの基礎的な原則

### 第1章 組み込みクラスを最大限に活用する
- 1.1 いつ組み込みクラスを使うべきか
- 1.2 true、false、nilを活用する
- 1.3 用途に応じて数値型を使い分ける
- 1.4 シンボルと文字列の違いを理解する
- 1.5 配列、ハッシュ、セット（集合）を使い分ける
- 1.6 Structを活用する（Structはもっと評価されるべき）

### 第2章 役に立つ独自クラスを設計する
- 2.1 組み込みクラスを継承するよりも委譲を使う
- 2.2 柔軟なインターフェースを設計する
- 2.3 状態を持つオブジェクトを最小限にする
- 2.4 クラスの責務を明確にする

### 第3章 変数を適切に使う
- 3.1 スコープを最小限にする
- 3.2 適切な名前を付ける
- 3.3 定数の使い方を理解する

### 第4章 メソッドと引数
- 4.1 メソッドの可視性を理解する
- 4.2 明確な引数にはキーワード引数を使う
- 4.3 真偽値の引数を避ける
- 4.4 オプション引数のデフォルト値にnilを使う

### 第5章 エラーに対処する
- 5.1 戻り値か例外か
- 5.2 独自の例外クラスを定義する
- 5.3 適切なレベルで例外を捕捉する
- 5.4 一時的なエラーには指数関数的バックオフで対処する

### 第6章 コードを読みやすくフォーマットする
- 6.1 一貫したスタイルガイドに従う
- 6.2 自己記述的なコードを書く
- 6.3 コードの読みやすさを追求する

### 第11章 きちんとコードが動くことをテストする
