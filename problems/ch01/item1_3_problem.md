# 第1章 1.3: 用途に応じて数値型を使い分ける

## 問題：割り勘の悪夢

あなたは割り勘アプリのバックエンド `BillSplitter` クラスを作成しています。
合計金額（税抜）と人数を受け取り、一人当たりの支払額（税込）を計算します。

しかし、現在の実装には2つの問題があります：
1.  **浮動小数点誤差**: 計算結果に `.00000001` のような端数が生じることがある。
2.  **不正確な初期化**: `BigDecimal` を使おうとしているが、初期化の仕方が間違っていて誤差が残っている。

### 課題

1.  テストを実行して失敗を確認する：`bundle exec rspec spec/ch01/item1_3_spec.rb`
2.  `lib/ch01/item1_3.rb` を分析し、なぜ誤差が出るのか特定する。
3.  以下の修正を行う：
    - `BigDecimal` の正しい初期化方法（文字列を使う）に変更する。
    - 割り算の結果が割り切れない場合（例: 100円を3人で割る）の端数処理を適切に行う（切り捨て、切り上げ、四捨五入のいずれかを選択し、テストを通す）。今回は **切り捨て (floor)** とします。

### ヒント (OSSからの学び)
`money` gem では `BigDecimal('10.5')` のように必ず文字列で初期化しています。
`BigDecimal(1.1)` と書いてしまうと、Floatの `1.1`（実際には `1.1000000000000000888...`）をそのまま高精度で保存してしまうため、意味がありません。
